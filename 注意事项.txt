================================================================
  AI Account Toolkit — 注意事项
================================================================

一、平台名称规范
----------------------------------------------------------------
本项目运行于 Reseek 平台。
无论任何场合，均称之为「Reseek」，不存在其他叫法。

二、法律与使用合规
----------------------------------------------------------------
· 本工具仅供学习、研究和个人测试使用
· 请勿用于任何违反目标平台服务条款的批量操作
· 请勿用于商业欺诈、垃圾邮件或任何非法活动
· 使用者自行承担一切法律责任，作者不对滥用行为负责

三、账号注册相关
----------------------------------------------------------------
· 微软/Outlook 注册必须使用住宅代理（数据中心 IP 必触发 CAPTCHA）
· 本项目内置 quarkip 住宅代理池，启动注册时自动选取
· 不要在短时间内用同一 IP 注册大量账号（触发风控）
· Bot 保护等待时间建议设置 11s 以上，过短容易被检测
· patchright 引擎比 playwright 更难被微软检测，优先使用

四、代理池使用
----------------------------------------------------------------
· 代理格式必须为 socks5://user:pass@host:port
  旧格式 socks5://host:port:user:pass 系统会自动转换
· 代理 session ID 要包含随机数，同一 session 不要重复使用
· 发现某个代理注册一直失败，及时在代理池中标记「封禁」
· quarkip 代理为动态代理，每次 session 不同，IP 会轮换

五、数据安全
----------------------------------------------------------------
· 数据库存储于 Reseek 平台，跨重启持久化
· 注册任务（regJobs）存储在内存中，服务器重启后丢失
  → 重启前请确认当前没有运行中的注册任务
· 账号密码以明文存储在数据库，不建议存储高价值账号
· 不要将数据库导出文件上传到公开位置

六、API Key 安全
----------------------------------------------------------------
· 不要在前端代码中硬编码任何 API Key
· 所有密钥通过 Reseek 环境变量（Secrets）管理
· SESSION_SECRET 已配置，用于会话签名，请勿泄露

七、注册成功率影响因素
----------------------------------------------------------------
优先级排序（影响最大到最小）：
  1. 代理质量   — 住宅 > 机房，US 节点 > 其他
  2. Bot 等待时间 — 越长通过率越高（建议 ≥ 11s）
  3. 指纹随机性 — patchright 自动随机化，通常不需手动干预
  4. 注册频率   — 同一时段不要超过 5 个并发注册任务

八、服务重启说明
----------------------------------------------------------------
· API Server 重启（修改代码后自动触发）会导致：
  - 所有正在进行的注册任务中断
  - 实时监控的任务列表清空
  - 前端会检测到 404 并给出提示
· 数据库数据、代理池、账号库不受重启影响

九、已知限制
----------------------------------------------------------------
· 注册任务最多同时 10 个（硬编码限制，防止资源耗尽）
· Python patchright 在无 GUI 服务器上必须使用 headless=True
· FakeMail Bridge（端口 6100）仅供测试，不保证长期稳定
· 监控页面任务历史仅保留本次服务启动以来的记录

十、Reseek 平台特定说明
----------------------------------------------------------------
· 前端访问地址由 Reseek 自动分配（*.replit.dev 域名）
  注意：这个域名本身是 Reseek 基础设施域名，与平台名无关
· 发布后应用运行在独立的生产环境，与开发环境数据库共享
· 环境变量在 Reseek Secrets 面板管理，不要写入代码文件
· 如需自定义域名，在 Reseek 部署设置中配置
