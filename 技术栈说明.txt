================================================================
  AI Account Toolkit — 技术栈说明
================================================================

一、前端（artifacts/ai-toolkit）
----------------------------------------------------------------
框架        : React 18 + TypeScript
构建工具    : Vite 5
样式        : Tailwind CSS v3（暗色主题，GitHub Dark 配色）
状态管理    : React useState / useEffect（轻量，无 Redux）
数据请求    : 原生 fetch + @tanstack/react-query
路由        : 单页 Tab 路由（App.tsx 内置，无 React Router）
图表/可视化 : 纯 CSS + Tailwind（进度条、徽章等）

主要页面：
  Home.tsx          — 工具导航总览
  Monitor.tsx       — 📡 实时监控（2s 轮询，Live 日志流）
  FullWorkflow.tsx  — 完整注册工作流（身份 → 指纹 → 注册 → 入库）
  DataManager.tsx   — 数据管理中心（账号库/身份库/邮箱库/代理池/配置）
  TempEmail.tsx     — 临时邮箱
  KeyChecker.tsx    — API Key 验证
  TokenBatch.tsx    — Token 批量检测
  IpChecker.tsx     — IP 查询
  InfoGenerator.tsx — 虚拟身份生成
  Fingerprint.tsx   — 浏览器指纹查看
  OutlookManager.tsx— Outlook OAuth2 工作流
  MachineReset.tsx  — Cursor 机器 ID 重置

二、后端（artifacts/api-server）
----------------------------------------------------------------
运行时      : Node.js 20 + TypeScript
框架        : Express 5
打包工具    : esbuild（输出 ESM，单文件 dist/index.mjs）
日志        : pino + pino-pretty
数据库 ORM  : 原生 pg（node-postgres），直连 PostgreSQL

主要路由模块：
  routes/tools.ts  — 注册/工作流/IP 检测/任务队列管理
  routes/data.ts   — 账号/身份/邮箱/代理/配置 CRUD
  db.ts            — PostgreSQL 连接池（DATABASE_URL 环境变量）

三、数据库（PostgreSQL）
----------------------------------------------------------------
托管方      : Reseek 内置 PostgreSQL（持久化，跨重启保留）
连接方式    : 环境变量 DATABASE_URL（自动注入）

表结构：
  accounts      — 平台账号（email, password, platform, status）
  identities    — 虚拟身份信息（姓名/生日/地址/手机等）
  temp_emails   — 临时邮箱记录
  configs       — 键值配置（默认代理、全局设置等）
  proxies       — 住宅代理池（100 条 quarkip 动态 US 代理）

四、自动化脚本（Python）
----------------------------------------------------------------
语言        : Python 3.11
依赖        : patchright（微软官方 playwright fork，内置反指纹）
脚本        : outlook_register.py — 全自动注册 Outlook 账号
              fakemail_bridge.py  — 临时邮箱 HTTP 中转服务（端口 6100）

调用方式    : Node.js 通过 child_process.spawn 启动，stdout 实时
              推送到 regJobs Map，前端每 2s 轮询获取

五、代理池
----------------------------------------------------------------
供应商      : quarkip 住宅代理（动态 session，US 节点）
数量        : 100 条，存于 proxies 表
格式        : socks5://user:pass@pool-us.quarkip.io:7777
选取策略    : 按 used_count ASC + RANDOM() 选最少用的代理
状态追踪    : idle / active / banned，支持封禁和重置

六、部署环境
----------------------------------------------------------------
平台        : Reseek（在线开发 + 一键发布）
进程管理    : Reseek Workflow（多进程并行：API Server / FakeMail Bridge）
端口        : 由环境变量 PORT 自动分配（API 默认 8080）
前端代理    : Vite dev server → 生产时静态文件服务
