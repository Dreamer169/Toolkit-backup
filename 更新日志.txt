[2026-04-11] v1.5.0 — 实时监控中心
----------------------------------------------------------------
新增
  + 新页面「📡 实时监控」（Monitor.tsx），2s 自动轮询
  + API 端点 GET /api/tools/jobs — 列出所有注册任务摘要
  + 监控页功能：API 健康/延迟、任务队列、实时日志流、
    最近入库账号表、代理池状态进度条
  + 支持在监控页直接停止运行中的注册任务
  + 支持暂停/恢复自动刷新、手动立即刷新

修复
  ! 日志轮询崩溃 —— API 返回 { type, message } 格式，
    前端误读 l.text（undefined）导致 classifyLine() 报错
    TypeError: Cannot read properties of undefined ('toLowerCase')
  ! Invalid hook call 连带错误（由上述崩溃引发）
  ! since 索引追踪逻辑错误（改用 d.nextSince，移除不存在的 l.offset）

调整
  ~ 导航栏顺序调整：实时监控、完整工作流、数据管理中心排在前列

[2026-04-10] v1.4.0 — 代理池自动接入注册流程
----------------------------------------------------------------
新增
  + 注册端点新增 autoProxy 参数，无手动代理时自动从池中选取
  + 代理选取策略：used_count ASC + RANDOM()，均衡轮询
  + 注册日志显示 [代理池自动选取] 标记，密码字段脱敏 ****
  + 完整工作流页显示代理池实时状态（绿色提示条、数量）
  + 「手动选取查看」按钮：预览将使用的代理地址
  + 启动按钮文字动态提示：「代理池自动选取」/ 「无代理」

[2026-04-09] v1.3.0 — 100 条住宅代理导入 + 代理 CRUD
----------------------------------------------------------------
新增
  + proxies 表（status, used_count, last_used, formatted）
  + 批量导入 100 条 quarkip US 住宅代理（socks5 格式）

修复

    原来：无限 pending；修复后：检测到 404 立即停止并提示

[2026-04-07] v1.1.0 — 完整注册工作流
----------------------------------------------------------------
新增
  + FullWorkflow.tsx — 端到端向导：
    身份生成（randomuser.me API）→ 浏览器指纹展示
    → Outlook 自动注册 → 自动入库
  + GET  /api/tools/workflow/prepare — 一步拿到身份+指纹+邮箱
  + POST /api/tools/outlook/register — 异步启动注册，返回 jobId
  + GET  /api/tools/outlook/register/:jobId — 轮询任务状态
  + DELETE /api/tools/outlook/register/:jobId — 停止任务
  + 「仅保存凭据」旁路：无代理时跳过注册直接入库
